What is Deno.js

Deno is a JavaScript and TypeScript runtime built on V8 (Google's JavaScript engine) and Rust. It was created by Ryan Dahl — the original creator of Node.js — and first released in 2018. Deno was designed to address design decisions in Node.js that Dahl considered mistakes in hindsight.

The name Deno is an anagram of Node. Unlike Node.js, Deno runs TypeScript natively without a compilation step, uses ES modules instead of CommonJS, imports dependencies via URL instead of npm, and has a built-in permission model that prevents access to file system, network, and environment by default.

// Run Deno without npm, without node_modules // Install: curl -fsSL https://deno.land/install.sh | sh // hello.ts — TypeScript runs natively const message: string = “Hello from Deno.js!”; console.log(message); // deno run hello.ts → Hello from Deno.js! // Import from URL — no npm install needed import { serve } from “https://deno.land/std@0.220.0/http/server.ts”; serve((req) => new Response(“Hello World”), { port: 8000 }); // deno run –allow-net server.ts

Deno 2.0 (released 2024) introduced npm compatibility — you can now import npm packages with npm:package-name syntax, making migration from Node.js easier while keeping Deno's security model.

Deno permission model

Deno's most distinctive feature is its security-first permission model. Every Deno script runs in a sandbox by default — it cannot access the file system, make network requests, read environment variables, or spawn subprocesses unless you explicitly grant those permissions via command-line flags.

Deno built-in features

Native TypeScript

Run .ts files directly. No tsc, no ts-node, no tsconfig.json required. Deno transpiles TypeScript internally using swc. Type-check with deno check file.ts.

Built-in Test Runner

deno test runs tests without Jest, Mocha, or Vitest. Uses Deno.test() with built-in assertion library from @std/assert. Supports async tests and subtests natively.

Formatter & Linter

deno fmt formats code (no Prettier needed). deno lint runs a built-in linter with configurable rules. Both are configured in deno.json.

Standard Library

Deno has a maintained standard library at jsr:@std covering HTTP, path, datetime, assert, collections, streams, CSV, YAML, and more — all reviewed and typed.

Web-Compatible APIs

Deno implements browser-compatible APIs: fetch, Request, Response, URL, TextEncoder, crypto, localStorage, Web Workers. Code written for Deno often runs in browsers too.

JSR Package Registry

JSR (JavaScript Registry) is Deno's modern package registry, supporting TypeScript natively and working across Deno, Node.js, Bun, and browsers. Import with import {} from "jsr:@scope/package".

Frequently Asked Questions
Deno is a secure JavaScript and TypeScript runtime built on V8 and Rust, created by Ryan Dahl. It runs TypeScript natively, has a built-in permission model (no file/network/env access by default), uses ES modules and URL-based imports instead of node_modules, and ships with built-in tooling for testing, formatting, and linting. Deno 2.0 added npm package compatibility.
Key differences: Deno is secure by default — scripts need explicit --allow-read, --allow-net etc. to access resources. Deno runs TypeScript natively without tsc. Deno uses ES modules (import/export) exclusively, not CommonJS (require). No node_modules — dependencies are cached globally. Deno ships a standard library, formatter, linter, and test runner. Node.js has a much larger ecosystem and better legacy compatibility.
Use Node.js when: you need a large npm ecosystem, you're maintaining existing Node.js code, your team knows Node.js well, or you need maximum package compatibility. Use Deno when: security matters (sandboxed execution by default), you want TypeScript without a build step, you're building edge/serverless functions (Deno Deploy), or you want a batteries-included runtime without configuring ESLint, Prettier, Jest separately. Deno 2.0 significantly closed the compatibility gap with npm.
macOS/Linux: curl -fsSL https://deno.land/install.sh | sh. Windows (PowerShell): irm https://deno.land/install.ps1 | iex. Homebrew: brew install deno. Verify with deno --version. Run a TypeScript file: deno run file.ts. Run with network access: deno run --allow-net server.ts.
Fresh is the official full-stack web framework for Deno, designed for Deno Deploy edge deployments. It uses islands architecture — all rendering is server-side by default, with optional client-side hydration for interactive components. No build step, no bundler, instant startup. Uses JSX/TSX with Preact. Fresh is the Deno equivalent of Next.js for Node.js.
